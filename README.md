# 赋语鸭

面向无言语/低言语能力人群的 AI 情境沟通助手。

> 帮助无法顺畅说话的人，在餐厅、医院、课堂、电话、紧急情况中，用 AI 快速生成自然、有情绪、符合本人风格的表达。

---

## 产品定位

### 一句话描述
不只是文字转语音，而是一个 AI 沟通代理：理解用户的关键词、当前场景和情绪，把碎片化输入转成自然、准确、有尊严的表达。

### 核心差异化

| 维度 | 传统 AAC | 赋语鸭 |
|------|----------|--------|
| 输入方式 | 逐字点选图片/词板 | 关键词输入、场景按钮、碎片词 |
| 输出质量 | 机械、僵硬、无情绪 | 自然、有语气、可切换风格 |
| 场景适配 | 通用词库，翻页找词 | 场景驱动，自动推荐高频表达 |
| 目标用户 | 偏重医疗/康复场景 | 日常社交、出行、餐厅、紧急 |
| 使用成本 | 专用设备昂贵、重 | 轻量移动端，手机即可 |

## 目标用户

### A 类：完全或几乎无法发声的人
脑瘫、ALS/渐冻症、喉切除术后、气管切开、部分自闭症非口语者。

痛点：传统 AAC 速度太慢，跟不上真实对话节奏。

### B 类：能发声但别人听不懂的人
构音障碍、口吃、声带疾病、帕金森、脑卒中后语言障碍。

痛点：说话别人听不懂，重复多次后容易放弃沟通。

### C 类：失语症（Aphasia）人群
知道自己想说什么，但找不到词或说不完整。美国每年新增约 18 万失语症患者。

痛点：句子碎片化、找词困难、说错词。

## 核心功能

### 功能 1：沟通身份卡（首页）
用户打开 App 第一眼看到：

```
我不能用声音说话，但我能理解你。
请给我一点时间。
请一次问一个问题。
```

解决第一秒的社交误解——告诉陌生人如何与用户沟通。

### 功能 2：场景模式
用户选择当前场景，App 自动展示该场景的高频表达按钮。

| 场景 | 示例按钮 |
|------|----------|
| 医院 | "我这里疼"、"我对这个药过敏"、"请联系我的家人" |
| 餐厅 | "我想点这个"、"不要辣"、"可以帮我打包吗" |
| 交通 | "我要去这里"、"请帮我打车"、"我坐过站了" |
| 学校/工作 | "我需要帮助"、"我没听清楚"、"请重复一遍" |
| 紧急情况 | "我需要去医院"、"请联系紧急联系人"、"我迷路了" |

### 功能 3：关键词变完整句（AI 核心功能）
用户输入几个碎片词，AI 自动生成完整的自然表达。

| 用户输入 | AI 生成的句子 |
|----------|---------------|
| "头 痛 药" | "我头很痛，可以给我止痛药吗？" |
| "水 冷 喝" | "我想喝一杯冰水。" |
| "不要 碰 我" | "请不要碰我，我需要一点空间。" |
| "饿 吃" | "我饿了，这里有我能吃的东西吗？" |

### 功能 4：语气改写
同一句话可一键切换表达风格。

| 语气 | "我不想要" 的改写 |
|------|------------------|
| 礼貌版 | "谢谢你，但我不需要。" |
| 朋友版 | "不用了，谢啦～" |
| 紧急版 | "请不要给我这个，我不能要。" |
| 强硬版 | "请停止，我已经明确拒绝了。" |

### 功能 5：离线应急卡
不依赖 AI 的本地固定模块，包含：
- 紧急联系人信息
- 过敏史与病史
- 家庭地址
- 常用药物
- SOS 一键语音播报

### 功能 6（可选）：双向沟通辅助
对方讲话 → 语音识别转文字 → AI 推荐回复建议。

## 技术架构

### 技术栈

| 层 | 技术 |
|----|------|
| 框架 | Next.js 16+ |
| 样式 | Tailwind CSS + shadcn/ui |
| AI | LLM（OpenAI API / 本地 Mock） |
| 语音 | 浏览器 SpeechSynthesis / 第三方 TTS |
| 部署 | Vercel |

### 页面结构（共 4 个核心页面）

```
/                 → 首页：沟通身份卡
/scenes           → 场景选择页
/scenes/[id]      → AI 表达生成页（关键词输入 + 生成结果）
/emergency        → 离线应急卡 / SOS
```

### 组件树

```
<App>
  <CommunicationCard />      // 首页沟通卡
  <SceneGrid />              // 场景选择网格
  <SceneDetail>
    <SuggestionButtons />    // 场景高频按钮
    <KeywordInput />         // 碎片关键词输入
    <GeneratedSentences />   // AI 生成的句子列表
    <ToneSelector />         // 语气选择器
    <VoicePlayButton />      // 语音播放
  </SceneDetail>
  <EmergencyCard />          // 离线应急卡
</App>
```

## 数据模型

### Scene（场景）

```typescript
interface Scene {
  id: string
  name: string          // "医院"
  icon: string          // emoji 或 icon 名称
  description: string   // 简短描述
  prompts: string[]     // 高频句子列表
}
```

### GeneratedSentence（AI 生成句）

```typescript
interface GeneratedSentence {
  id: string
  text: string         // 生成的句子
  tone: Tone           // 语气
  emoji?: string       // 可选的表情辅助
}
```

### Tone（语气）

```typescript
type Tone = 'polite' | 'casual' | 'urgent' | 'firm' | 'professional'
```

### EmergencyInfo（应急信息）

```typescript
interface EmergencyInfo {
  name: string
  emergencyContact: string
  emergencyPhone: string
  allergies: string[]
  conditions: string[]      // 病史
  address: string
  medications: string[]
}
```

## 设计原则

1. **尊重感优先**：不使用幼稚图标、儿童化配色。主色调选用冷静、专业的蓝色系。
2. **大字可读**：所有文字不小于 16px，核心信息不小于 24px。
3. **渐进呈现**：首页只有核心信息，不堆砌功能。用户按需深入。
4. **离线可用**：应急卡和沟通卡不依赖网络，预置内容本地加载。
5. **一键直达**：紧急场景入口常驻底部或首页，无需翻找。
6. **AI 透明**：AI 生成的句子用户可确认后再播放，不自动发声。

## 路演要点

### 产品故事
"现有 AAC 工具经常让用户一个字一个字地点选，速度慢、表达僵硬、缺少个人风格。我们的产品不是简单的文字转语音，而是一个 AI 沟通代理：它能理解用户的关键词、当前场景和情绪，把碎片化输入转成自然、准确、有尊严的表达。我们先从医院、餐厅、紧急情况等高频刚需场景切入，帮助无言语和低言语能力人群更快、更安全地参与真实生活。"

### 核心展示功能
1. 打开 App → 沟通卡示意
2. 选择场景 → 高频按钮
3. 输入碎片词 → AI 生成完整句
4. 切换语气 → 不同风格表达
